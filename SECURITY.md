# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in NetPilot, please report it privately. **Do not open a public GitHub issue.**

Send details to **security@netpilot.io** (or **contact@netpilot.io** if security@ is not available).

Include:
- A description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested mitigation

We will acknowledge your report within 2 business days and aim to provide a remediation timeline within 7 business days.

## Scope

The following are in scope for security reports:

- The NetPilot web application (app.netpilot.io)
- The NetPilot API (api.netpilot.io)
- The marketing site (netpilot.io)
- Authentication and session handling
- VM provisioning and isolation
- Data storage and encryption

## Out of Scope

- Social engineering attacks
- Physical attacks
- Denial-of-service testing
- Findings from automated scanners without proof of exploitability
- Third-party services (Clerk, Stripe, Cloudflare, GCP, Vercel, Neon)

## Recognition

We are happy to credit researchers who responsibly disclose vulnerabilities. Let us know how you'd like to be acknowledged.
